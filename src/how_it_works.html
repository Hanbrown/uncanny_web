We use Flask as a web server. Upon loading the homepage, a request is sent to the server to analyze the bowling alley photo.
Using the equations from the Orghidan, Salvi, et al. paper, we calculate the vanishing points and then the focal length.
We do calculate a position matrix, but we do not use it as we were not getting satisfactory results.
The server sends the focal length back as a JSON object.
A front-end JS script sets the ThreeJS camera as the focal length from the server, and the scene is rendered.
We add orbit controls so that the camera can be properly positioned and rotated within the scene.
